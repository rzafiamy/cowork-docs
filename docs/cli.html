CLI Reference

Cowork provides a comprehensive command-line interface for interaction, configuration, and auditing.

Core Concept: CLI vs. Slash Commands

Cowork utilizes two distinct mechanisms for command execution depending on where you are currently positioned in the terminal. Understanding the difference is crucial for mastering the agent.

1. System CLI Commands (cowork ...)

Executed directly from your bash/zsh terminal. Used to boot the agent, run one-shot background tasks, set up environments, or view global state without entering chat.

~ $ cowork run "fix tests"

2. Slash Commands (/...)

Executed inside the interactive cowork chat REPL. Used to modify the live state of the agent, switch models dynamically, outline goals, or review localized memory without leaving the conversation.

Cowork> /model claude-3.5-sonnet

System CLI Commands

Execute these directly in your external terminal.

cowork chat

Interactive

Purpose: Boots the interactive Agentic REPL.

The Logic: You want an ongoing pair-programming or research session where the agent retains rolling context of the conversation and can ask you clarifying questions.

Options:

  • --session-id, -s <id> : Resume a specific past session workspace.
  • --trace : Force persistence of a full workflow trace (.jsonl) for debugging.
  • --no-banner : Skip the ASCII dashboard for faster boot.
$ cowork chat --no-banner
Cowork> Ready.

cowork run "<task>"

One-Shot

Purpose: Executes a task start-to-finish without entering an interactive prompt, then exits.

The Logic: Ideal for CI/CD pipelines, quick detached fixes, or bash aliases where human interaction is unnecessary or impossible.

Options:

  • --session-id, -s <id> : Run the discrete task using context from an existing session.
  • --model, -m <model> : Override the text model for this run specifically.
  • --no-stream : Disable stdout generation streaming completely.
  • --trace : Generate an execution audit trace on completion.
$ cowork run "Audit package.json" -m claude-3-opus
✅ Completed.

cowork trace

Audit

Purpose: Renders executed JSONL traces into a human-readable timeline.

The Logic: Traceability into the black box. Find exactly where token counts inflated or why a tool call failed by inspecting exact system inputs and outputs offline.

Options / Arguments:

  • <file_path> : (Optional) Explicit path to a `trace.jsonl` file. Defaults to latest globally.
$ cowork trace ~/.cowork/traces/session_123.jsonl
Step 1 [0.08s]: Booting meta-router... Step 2 [1.2s]: Tool Call -> codebase_search

cowork memory

Memoria

Purpose: Outputs global Memoria summary status and top graph edges.

The Logic: You want to verify what cross-session knowledge the agent is holding onto natively or diagnosing what preferences are bleeding over.

$ cowork memory
Memoria Graph Status: 45 nodes, 112 active edges. - (User, prefers_db, postgres) [Weight 0.9]

cowork sessions / cowork jobs / cowork cron

Purpose: Manage long-term environment orchestration elements including tasks and persisted chats.

The Logic: You need to reboot the server but you don't want to lose the context of the 3-hour pair programming session you had. Just look up the session and resume it. Or manage the built-in periodic cron jobs daemon.

  • cowork sessions (list): Lists all historical session objects.
  • cowork jobs: Displays Sentinel background job dashboard.
  • cowork cron list/view/run-pending: Interacts with the local scheduler daemon.
$ cowork sessions list
1. 8f2a9 - "Debugging Redis Cache" - 2.5MB context

cowork config / setup / reset

Purpose: Controls the `.cowork` host configuration variables and authentication state directly from the terminal without manual JSON editing.

The Logic: It's easier and safer to use CLI bindings to configure global system architecture settings rather than manually writing schema-sensitive settings.

  • cowork setup: Interactive wizard for bootstrapping initial env vars.
  • cowork config: Displays active system configurations and keys.
  • cowork tokens: Cumulatively monitor token consumption per exact model context.
  • cowork reset: WARNING: Purges the entire `~/.cowork` local state namespace.
$ cowork tokens
Model [claude-3.5-sonnet] : 1,402,112 in | 85,022 out

Interactive Slash Commands

Execute these directly at the Cowork> prompt during an active chat session.

/model <name>

Purpose: Instantly hot-swaps the underlying generative model.

The Logic: Swap between fast/cheap models vs slow/heavy ones mid-conversation. Purposely only updates core text models, preserving Vision/TTS endpoint mappings.

Cowork> /model claude-3.5-sonnet
Model updated to claude-3.5-sonnet.

/memory

Purpose: Interact directly with the local knowledge graph.

The Logic: Useful for querying the embedding memory to ensure the agent digested a standard operating procedure correctly before proceeding to execution.

  • /memory search <query>: Semantic vector search against graph knowledge.
  • /memory add <sub> <pred> <obj>: Manually inject EWA durable triplets.
  • /memory clear: Wipes long-term memory for total domain pivots.
Cowork> /memory search 'deployment pattern'
Retrieval Results: Docker Compose with GitHub Actions (Score: 0.88)

/acl log

Purpose: Streams live output from the file permission gatekeeper.

The Logic: Useful for visually tracking which exact files the FileManager is modifying in the background and catching unauthorized directory jumps.

Cowork> /acl log
[ACL] PERMIT: read -> /etc/hosts

/trace

Purpose: Inspects the context block tracing metadata linked to this actual live session buffer.

The Logic: If an agent unexpectedly fails inside a loop, you can immediately dump its reasoning steps without having to open the session JSONL file manually.

  • /trace full: Expands timeline view to show full textual payloads.
  • /trace raw: Displays unformatted JSON event drops.
  • /trace path: Returns the exact absolute path to the current trace.
Cowork> /trace path
Trace file located at: ~/.cowork/traces/session_abc.jsonl

/scratchpad & /workspace

Purpose: Query the local contextual memory stores.

The Logic: View what parameters the gatekeeper truncated into side-storage (/scratchpad) versus the direct OS files the agent has successfully observed (/workspace).

  • /scratchpad: Lists elements offloaded by the Gatekeeper to save context space.
  • /workspace: View current workspace directory maps and artifacts.
Cowork> /scratchpad
1 Item: [server_crash_log.txt] mapped into context block.

/jobs

Purpose: Adjust background task queues.

The Logic: Act on the Sentinel container pool items directly inside the chat loop if one fails or hangs.

  • /jobs clean: Purge success/failure job artifacts from Sentinel queue.
  • /jobs resume <job_id>: Reboot a dead job container within the agent queue.

/config set <key> <value>

Purpose: Override strict configurations safely.

The Logic: Adjust variables like temperature_agent natively without requiring the server node to restart or losing active thread context.

Cowork> /config set max_steps 25
max_steps set to 25 globally.

/help, /new, /load <id>, /clear, /exit

Purpose: The core lifecycle routers.

The Logic: Utilities to branch, fold, and discard system interactions cleanly.

  • /new: Borks a clean context window instantly without navigating bash.
  • /load <id>: Fetches an archived JSON session history directly into the live thread.
