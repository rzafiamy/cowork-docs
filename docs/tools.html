Tools Registry

Tools are the primary way Cowork agents interact with the outside world. They are modular, schema-driven, and governed by the security firewall.

Tool Architecture

All tools in Cowork inherit from a common BaseTool class and are registered in a central registry. This ensures consistent error handling, metadata extraction, and safety checks.

Built-in Tools

Core capabilities like File System access, Scratchpad management, and CLI execution that are compiled into the Cowork binary.

External Adapters

Dynamic wrappers for 3rd-party APIs (Search, Weather, News) that map external schemas to the Cowork internal protocol.

Execution Gateway

Every tool execution passes through the ExecutionGateway. This singleton:

  • Identifies the tool by its unique name.
  • Validates the input arguments against the tool's JSON schema.
  • Consults the Firewall to determine if the user needs to confirm the action.
  • Captures the output and any execution errors for the trace log.

CLI Tool Management

# List all available and activated tools
cowork tools

Tool Categories

Tools are grouped into categories, which allows the Meta-Router to activate only relevant toolsets, reducing prompt tokens and improving reasoning accuracy.

FILE_SYSTEM

Read, write, and list project files.

SCRATCHPAD

Manage episodic pass-by-reference memory.

BASH_COMMANDS

Execute Linux shell commands via the CLI.

WEB_SEARCH

Dynamic Google/Brave search integration.
