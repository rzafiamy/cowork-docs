The Sentinel (Job Queue)

Cowork uses a central job management system called The Sentinel to track concurrent executions, enforce resource limits, and provide a durable audit trail of all agent actions.

Execution Lifecycle

Every time an agent turn is initiated (via CLI, Cron, or Web), it is registered as an AgentJob. This allows the system to monitor its progress and handle unexpected interruptions.

01

Registration

The job is assigned a unique 8-character ID. The Sentinel checks global concurrency limits (default: 10 concurrent jobs).

02

Trace Binding

A dedicated .jsonl trace file is created and linked to the job ID for real-time observability.

03

Loop Execution

The agent enters its reasoning/action loop. Token usage and tool calls are tracked per-job.

04

Completion/Failure

The final result or error message is persisted. Ghost jobs (interrupted processes) are automatically detected and marked as failed.

Durable Job History

All jobs are persisted to ~/.cowork/jobs.json. This history can be inspected to understand past agent behavior and resource consumption.

CLI Commands

# Show the jobs dashboard (recent 24 jobs)
cowork jobs
# Wipe all job history
cowork jobs clean

Ghost Job Detection

Because Cowork is a CLI-first tool, a process might be killed (SIGINT/SIGTERM) during an agent turn. The Sentinel identifies jobs marked as "RUNNING" during startup and re-labels them as "Ghost jobs" with an appropriate error message, ensuring the concurrency slots are freed.
