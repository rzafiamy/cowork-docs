Issue Manager

The Issue Manager is a specialized knowledge base that stores failure patterns and their solutions, allowing Cowork to learn from errors and provide real-time hints to the LLM during future turns.

Learning from Failure

When a tool call fails or a specific technical hurdle is encountered, a "Problem Triplet" can be recorded. This triplet consists of:

The Issue

A description of what went wrong (e.g., "Connection Timeout on port 8080").

The Reason

The underlying cause (e.g., "The local dev server was not started").

The Solution

Steps to fix it (e.g., "Run 'npm run dev' before initiating the sync tool").

Vector-Based Retrieval

Issues are stored in a SQLite database with vector search capabilities (via sqlite-vec). When the agent encounters an error message in any session:

  1. The system localizes the error text and generates a temporary embedding.
  2. It queries the issue_vec virtual table for high-similarity triplets.
  3. If a match is found (similarity > 0.5), the solution is injected into the agent's current scratchpad or context as a "troubleshooting hint."

CLI Commands

# List all known issues
cowork issues list
# Add a new issue-solution triplet manually
cowork issues add "API Auth Error" "Expired Token" "Run 'login' again"
# Clear the issue database
cowork issues clear

Automatic Recording

The record_issue_solution tool allows the agent itself to contribute to this knowledge base. If an agent successfully debugs a problem, it can proactively save the "Reason" and "Solution" for future instances.

Storage Location

All issue data is stored in a SQLite database at:

~/.cowork/issues/issues.db
