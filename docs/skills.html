Available Skills & Tools

Cowork uses a modular skill system to bridge the gap between "knowing how to think" and "knowing how to act."

Why Skills?

While Large Language Models (LLMs) are powerful reasoners, they suffer from context bloat and hallucination when presented with hundreds of tools at once. Skills solve this by:

  • Dynamic Context Management: Only relevant tools and instructions are injected into the agent's prompt based on the detected intent.
  • Scoped Reasoning: Skills provide domain-specific "workflows" and "guardrails" that guide the agent's strategy.
  • Security Isolation: Tools are gated by Trust Tiers, ensuring that sensitive operations (like code execution) require explicit user authorization.

Anatomy of a Skill

Every skill is defined by a SKILL.md file using a YAML frontmatter for metadata and standard Markdown for instructions. 

---
name: coding-tools
description: Inspect, edit, and validate codebase changes safely.
triggers: [code, implement, bug, refactor]
trust_tier: 3
tool_categories: [CODING_TOOLS]
---
# Instructions
Workflow: 1. Read file 2. Edit 3. Test...
Guardrails: Never delete files without confirming...

Description

The "Sales Pitch" for the Router.

Crucial for the Meta-Router to understand when to activate this skill during a turn.

Triggers

Semantic Keywords.

Helps index the skill for rapid retrieval when specific concepts (like "bug" or "git") are mentioned.

Trust Tier

The Security Level (1-3).

Tier 1 is purely informational. Tier 3 (Code/BASH) triggers strict firewall checks.

Workflows

The "Chain of Thought".

Embedded in the Markdown body to provide the agent with a proven recipe for task completion.

What defines a "Good Skill"?

  • High Specificity: A good skill does one thing (e.g., "YouTube Search") exceptionally well.
  • Narrow Permissions: It only requests the exact tools needed to fulfill its description.
  • Negative Constraints: It includes explicit "Never do X" guardrails to prevent agent overreach.

Skill Categories

Skills in the Cowork library are organized into semantic categories:

Development

Repository management, automated testing, and code refactoring. Designed for direct manipulation of the workspace.

Research

Web crawling, market intelligence, and news extraction. Focused on gathering external data for synthesis.

Data Services

Integration with persistent stores like Supabase and Nextcloud. Bridges ephemeral agent turns with durable storage.

Productivity

Organizers, scheduling, and habit tracking. Handles time-sensitive or task-oriented workflows like Cron monitoring.

System

Low-level operating system hooks, memory management, and scratchpad utilities. The foundation for all other skills.
