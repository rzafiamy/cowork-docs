Core Algorithms

Deep dive into the deterministic logic that powers Cowork's reliability and token efficiency.

Input Gatekeeper

The Gatekeeper algorithm scores user input to decide if a request can be "fast-pathed" or requires the full heavy-weight agentic pipeline.

Keyword Probability Scoring

Input is matched against high-entropy action terms. If probability > 0.75, the full router is engaged.

Token Density Analysis

Inputs exceeding 2,500 tokens are offloaded to a scratchpad and replaced with a reference key.

Meta-Routing Logic

Manages dynamic skill activation, ensuring the LLM only sees the tools relevant to the current task.

Classification

Zero-temperature intent classification and intent hierarchy resolution.

Pruning

Schema filtering that prunes 90% of available tools to prevent model confusion.

Context Compression

Map-Reduce sequence to preserve linear reasoning without hitting context limits.

Workflow Execution Planning

Cowork uses a Pre-Execution Roadmap (PER) algorithm to ensure tasks are completed without infinite loops or "reasoning drift."

1 Linear Reasoning Chain

Before the first tool is called, the agent must generate a JSON-structured plan. This plan identifies dependencies between tools (e.g., "I must read the file before I can grep for the pattern").

2 Persistence Verification

Every tool execution is checked against the plan's current step. If the output of a tool significantly deviates from the plan's expectation, the agent is forced to "re-route" and regenerate the plan from that point.

3 Termination Anchors

The plan includes explicit "Exit Success" and "Exit Failure" conditions. The loop terminates immediately when these anchors are met, preventing unnecessary token spend on "wrap-up" reasoning.

The REACT-Sentinel Loop

The core execution loop is a modified REACT (Reasoning + Acting) pattern governed by The Sentinel:

// Pseudo-code for Core Loop
while (steps < max_steps) {
thought = agent.think(state, plan);
tool_call = thought.extract_tool();
if (firewall.authorize(tool_call)) {
result = tool_call.execute();
state.update(result);
}
if (plan.is_met(state)) break;
}
