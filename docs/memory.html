Phase 07 • Memoria

The Memoria Engine

Cowork implements a hybrid long-term memory system designed for zero-latency local execution. It combines a structured Knowledge Graph with rolling session summaries to provide deep personal context without context drift.

Local SQLite

Zero external dependencies. Knowledge stays on your machine in a hardened SQLite database.

Vector KNN

Embedded semantic search using sqlite-vec for high-fidelity retrieval of relevant facts.

EWA Decay

Exponentially Weighted Average decay ensures that recent knowledge is prioritized.

Hybrid Knowledge Architecture

Memoria operates on two parallel tracks to balance long-term facts with immediate conversation context.

KG

Knowledge Graph (SPO Triplets)

Atomic facts stored as Subject-Predicate-Object triplets. Used for durable preferences, personality traits, and technical stack requirements.

USER | PREFERS | TAILWIND PROJECT | USES | PYTHON 3.12
SS

Rolling Session Summary

A compressed representation of the current conversation history. It provides short-term context and tracks active goals without bloating the prompt.

Retrieval Strategy

When a query is received, Memoria executes a multi-stage retrieval flow to find the most relevant context:

Vector KNN Search sqlite-vec
2

3
Keyword Overlap Fallback BM25-like
