File System & Internal Structure

Understand how Cowork manages state, memory, and security through its internal .cowork directory.

The .cowork Directory

The .cowork folder is the brain of the agent. Located by default in the user's home directory (~/.cowork), it persists all session data, configurations, and long-term memories.

Directory Breakdown

sessions/

Stores full state objects for every chat session. Files are named using UUIDs. These allow the agent to resume any conversation with perfect context retention.

traces/

Contains execution logs for every tool call and reasoning step. Organized by session UUID, with individual steps saved as timestamped JSONL files.

memoria/

The persistent storage for the Knowledge Graph and Vector Database. Contains SQLite (.db) files with semantic triples and embeddings.

scratchpad/

Used for context offloading. When a conversation exceeds token limits, older chunks are moved here to keep the "working memory" lean.

api_cache/

Caches LLM and tool responses to reduce latency and token costs. Essential for debugging and re-running failed steps without extra expense.

workspace/

A managed containment area for temporary files, git clones, and build artifacts generated during task execution.

Core Configuration Files

File Purpose
config.json Global settings, model preferences, and API endpoints.
firewall.yaml Security policies and tool permission manifests.
tokens.json Cumulative usage tracking and model-specific cost analysis.
jobs.json Registry of background and long-running autonomous tasks.
super_history.db Global search index for all inputs and outputs across every session.

Naming Conventions

Cowork uses strict naming conventions to ensure deterministic lookup and auditability:

  • UUIDs: Sessions and trace directories use version 4 UUIDs (e.g., 528840d4...) to avoid collisions in multi-agent environments.
  • ISO Traces: Trace files use concentrated ISO 8601 timestamps: YYYYMMDDTHHMMSSZ_shortid.jsonl.
  • Snapshots: Backups and migrations are prefixed with pre_migration_ and a compact date string.

The "Process" Lifecycle

During a high-complexity task, Cowork creates transient files to manage its reasoning. Understanding these helps in auditing the 7-Phase Protocol.

plan.json

Generated during Phase 04 (Planning). It contains the graph of steps the agent intends to take before it starts executing them.

context.jsonl

The ephemeral representation of the Execution Loop (Phase 05), tracking the current prompt state and tool outputs.
